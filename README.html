T3: Todo-list / Time / Tracker

This project is a command-line utility to keep track of your projects. It is written in Python3. I got the project from https://github.com/j-adamski/T3-Todo-List-Time-Tracker,  and made some changes. All the *.md files came from that project. The reason for not continuing with the original project is that some of the changes affect the format of the tasks.json file, so the two versions are not compatible.

A project is a list of tasks. A project can have only one task. Tasks have names, a timeSpent attribute, a timeStamp attribute, and a completed attribute. In terms of Python data structures, a task is a dictionary, with task attributes corresponding to keys.

To run the program, use

python3 t3.py arguments

At the moment, the following work (the first line assumes the command t3 runs t3.py)

t3 --report all
python3 t3.py --report all        # if not installed

lists all your projects and, within each project, the tasks. Internally, t3Report() is called.

t3 --add projectname task1 task2 ... taskN:

This creates the project and listed tasks. If no tasks are listed, one is created with name "maintask". If projectname already exists, then the tasks are added if they do not exist already. Any tasks that do already exist are ignored.

All data is stored in a file tasks.json, in directory tasks. (This is awkward; eventually the tasks.json should be in a designated place relative to the user's "home" directory.) If tasks/tasks.json is not found, the program will abort with an error.

At the top level, this file is a directory with 't3' as a single key. So, if you read the json file into a Python data structure S, you have:

  • S['t3'] is the project dictionary: a map from project names to lists of tasks
  • If P is in S['t3'].keys(), then P is the string name of a project, and L = S['t3'][P] is a list of tasks
  • Each element of L is a dictionary representing a specific task, with L['taskname'] as the task's name
